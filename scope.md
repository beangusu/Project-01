# Project Scope Statement

**Project Name:** Personal Introduction Webpage

## Project Purpose

> *Why are we doing this project?*

The point of this project is to put the project management concepts from *Project Managing for the Unofficial Project Manager* into practice with a real, hands-on task! By figuring out the purpose, goals, and boundaries of the project before writing a single line of HTML, we are practicing the habit of planning first instead of just diving in and figuring it out as you go. The project will also bridge the gap between the project management side of the course and the more technical work with practical web development skills.

## Description

> *How would you describe this project?*

This project is a single-page personal introduction website built with semantic HTML!

This website will...

- Include a `<title>` element in the document `<head>`
- Display my name in an `<h1>` heading and my major *or* professional interests in an `<h2>` subheading
- Contain *at least* two `<p>` paragraphs describing my goals for this course and my career
- Feature an unordered list (`<ul>`) with *at least three* `<li>` items covering my hobbies *or* interests
- Link to my GitHub profile using an `<a>` tag

This `scope.md` file is also part of the submission, written in Markdown following the Project Scope Statement template from PMUPM Chapter 3. Both files will be submitted together as a single `.zip` archive.

## Desired Results

> *What specific outcomes must we achieve to be successful?*

- A valid `index.html` file that checks every structural box required:
  - `title`
  - `h1`
  - `h2`
  - at least two `p` elements
  - a `ul` with at least three `li` items
  - an `a` hyperlink to my GitHub profile
- A complete `scope.md` file written in Markdown that follows the PMUPM Chapter 3 Project Scope Statement template, with thoughtful entries for the following sections:
  - "Project Purpose"
  - "Description"
  - "Desired Results"
  - "Exclusions"
- Both files packaged into a single `.zip` file and submitted on time, meeting every minimum expectation listed in the assignment guidelines

## Exclusions

> *What is out of the project's scope and should **NOT** be included?*

- Any CSS styling or visual design
- JavaScript or any interactive functionality
- Multiple pages or external assets (such as images or third-party frameworks)
- Backend components, databases, or any form of hosting or deployment
- Any content/features beyond what the assignment rubric explicitly requires
